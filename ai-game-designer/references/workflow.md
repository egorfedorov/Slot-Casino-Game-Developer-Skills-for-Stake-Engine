# Workflow

## 1. Vision And Constraints

- Define audience, session goals, and platform envelope.
- Define hard constraints (economy/fairness/compliance).

## 2. Loop Architecture

- Define core gameplay loop.
- Define progression/meta loop.
- Define retention loop and cadence.

## 3. System Specs

- Define resources, sources/sinks, and progression gates.
- Define feature list and dependency graph.

## 4. Integrity Validation

- Validate required sections and references.
- Validate economy/progression coherence.
- Validate risk constraints and telemetry coverage.

## 5. Handoff

- Produce implementation-ready mapping and acceptance checks.
- Include unresolved risks and follow-up recommendations.
