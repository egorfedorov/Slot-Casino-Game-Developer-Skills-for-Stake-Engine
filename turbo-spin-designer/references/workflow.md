# Workflow

## 1. Mode Definition

- Define normal, quick, and turbo speed targets.
- Define minimum phase durations for spin, stop, and resolve.

## 2. Stop Behavior

- Define stop behavior by state.
- Ensure stop never skips required outcome presentation.

## 3. UI Mapping

- Define button labels per state.
- Lock speed changes during resolve and payout.

## 4. Validation

- Validate timing budgets and readability.
- Validate event order and stop determinism.

## 5. Handoff

- Deliver spec, timing table, and QA checklist.
