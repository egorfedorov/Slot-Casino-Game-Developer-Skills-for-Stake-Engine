# UX Rules

## Control Rules

- Primary spin control must remain visually dominant in actionable states.
- Bet and balance must remain readable during gameplay.
- Controls disabled by state must communicate reason clearly.

## Responsive Rules

- Provide dedicated desktop and mobile layouts.
- Keep critical controls reachable by thumb zone on mobile.
- Avoid layout jumps during animation sequences.

## Feedback Rules

- Keep event feedback durations explicit.
- Prevent conflicting overlays in the same state window.
- Ensure modal dialogs do not trap gameplay state unintentionally.

## Accessibility Rules

- Minimum touch target size: `44px`.
- Support reduced motion mode.
- Ensure contrast-aware styling support in key panels/buttons.
