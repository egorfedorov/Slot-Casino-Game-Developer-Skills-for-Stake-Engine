# Workflow

## 1. Scope Definition

- Define target outcomes and hard constraints.
- Identify dependencies and ownership boundaries.

## 2. Plan

- Break work into deterministic steps.
- Define measurable checkpoints.

## 3. Execution

- Run incremental changes and collect evidence.
- Keep iteration history and config context.

## 4. Validation

- Validate key contracts and constraints.
- Stop on blocker violations.

## 5. Handoff

- Summarize outcomes, patch plan, and verification steps.
